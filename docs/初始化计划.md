## 二、近期目标（下一阶段）

### 阶段 1：真实落盘层（已完成）

- 已实现 `runtime_store.py`，支持项目状态、上下文包、任务更新结果落盘。
- `plan/context-pack/task-update` 已接入可追溯日志（项目级 + 会话级 `events.jsonl`）。
- `task-update` 已支持从 `execution-tasks.json` 回填历史状态，保证跨命令连续推进。

### 阶段 2：状态闭环（P0）

- 新增 `ProjectRuntime` 统一加载器，确保 runtime 成为唯一事实源。
- 从 `execution_tasks` 派生 `current_stage`，避免“任务机”和“阶段机”分叉。
- `task-update` 后同步写回：
  - `state.json`
  - `execution-tasks.json`
  - `project-plan.json`

### 阶段 3：上下文权限与阶段门禁（P0/P1）

- `context-pack` 按 `consumer_roles` 严格过滤上下文。
- `stage_allowed=false` 默认阻断，只有显式允许时才跳过。
- `context-pack` 改为先加载 runtime，再构造 packet。

### 阶段 4：输入摄取层

- 接入 PDF 解析与笔记归档。
- 产出 `research.summary`、`glossary`、`formula.catalog` 的真实内容生成器。

### 阶段 5：执行适配层

- 对接 HTML / Manim / SVG 子任务执行适配器。
- 统一子任务回传格式，写入长期与短期上下文。

### 阶段 6：控制台 MVP

- 把现有 CLI 编排能力包装成 Web 控制台。
- 第一版聚焦：
  - 项目创建
  - 阶段流展示
  - 任务看板
  - context packet 可视化
  - 手动任务推进

### 阶段 7：审核与后处理

- 实现 `review.outputs` 的自动化校验器。
- 接入配音、字幕、拼接流水线，并受审核关卡保护。
