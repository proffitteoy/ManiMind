# 第三方整合

第三方资产统一并入 `resources/`，不再使用独立 `vendor/` 目录。

## 目录落位

- HTML skill 与模板：`resources/skills/html-animation/`
- HyperFrames 精选参考：`resources/references/hyperframes/`
- Manim skill：`resources/skills/manim/SKILL.md`
- ClaudeCode 参考源码包（可选）：`ClaudeCode/`

## 白名单同步

使用 [sync-thirdparty-assets.ps1](/C:/Users/84025/Desktop/ManiMind/scripts/sync-thirdparty-assets.ps1) 同步，脚本只复制白名单内容，不复制完整上游仓库。

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\sync-thirdparty-assets.ps1
```

若源目录不在仓库根目录，可显式传入：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\sync-thirdparty-assets.ps1 `
  -HtmlSkillSource "<AI-Animation-Skill-main 的实际路径>" `
  -HyperframesSource "<hyperframes-main 的实际路径>"
```

同步完成后，允许删除外层开源仓库目录，只保留 `resources/` 内精简资产。

## ClaudeCode 抽取策略

- `ClaudeCode/` 仅作为“参考源码压缩包”，不属于运行时硬依赖。
- 可学习并落地的内容优先放入 `src/manimind/`：
  - 角色编排约束（只读探索、只读规划、验证专用角色）
  - 上下文装配策略（按角色和阶段注入、可缓存提示词分段）
  - 任务状态机与阻塞依赖（包含验证关卡）
- 抽取映射见 [ClaudeCode抽取清单.md](/C:/Users/84025/Desktop/ManiMind/docs/ClaudeCode抽取清单.md)。

## 维护原则

1. 只保留直接服务当前项目编排与渲染任务的资产。
2. `resources/` 下资产视为“裁剪后的第三方内容”，与 `src/manimind/` 自研编排逻辑分离。
3. `ClaudeCode/` 可删除，删除前需确认其可复用能力已在 `src/manimind/` 和文档内沉淀。
4. 白名单变更必须同步更新本文件与同步脚本。
