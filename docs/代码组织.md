# ManiMind 代码组织

## 总体原则

- 当前仓库只承担“编排层”和“项目层”职责。
- HTML 动画、视频渲染、Manim 数学动画优先复用外部能力目录，不在本仓库重复实现。
- 所有可持久化状态都必须落到明确目录，不允许隐藏在临时脚本或未命名文件中。
- 任何结构性改动都必须同步更新 `docs/architecture.canvas` 与对应 Markdown 架构文档。

## 目录约定

```text
docs/                     项目文档
docs/architecture.canvas  正式架构白板（Obsidian Canvas）
frontend/manimind-console/ Next.js 控制台骨架（前端层）
backend/                  FastAPI 接口骨架（API 层）
manim-worker-pov/         Manim Worker 独立 POC，不属于主编排 runtime
configs/                  项目配置与示例清单
scripts/                  PowerShell 初始化与检测脚本
src/manimind/             核心编排代码
tests/                    单元测试
runtime/projects/         长期项目上下文
runtime/sessions/         短期协作上下文
runtime/cache/            非关键缓存
outputs/                  最终输出与中间交付物
logs/                     运行日志
resources/skills/html-animation/   已并入的 HTML 动画 skill
resources/references/hyperframes/  已并入的 HyperFrames 参考资产
resources/skills/manim/SKILL.md    Manim skill 文档副本
```

## `src/manimind/` 模块边界

- `models.py`
  定义项目输入、镜头分段、任务分发表、上下文记录和审核检查点的数据模型。
- `workflow.py`
  定义默认阶段、子 Agent 分发规则、上下文蓝图和审核关卡。
- `bootstrap.py`
  负责仓库级初始化目录与外部依赖路径检测。
- `main.py`
  提供最小 CLI 入口，便于后续接入自动化命令。
- `context_assembly.py`
  负责按角色和阶段装配上下文包，并支持提示词分段缓存。
- `task_board.py`
  负责执行任务状态机、阻塞依赖检查与验证关卡提醒。
- `runtime_store.py`
  负责项目状态、上下文包、任务更新结果的落盘与审计日志写入。
- `runtime.py`
  负责 `ProjectRuntime` 统一加载、快照回填与 `current_stage` 派生。

## `backend/` 模块边界

- `main.py`
  FastAPI 应用入口与路由注册。
- `api/projects.py`
  项目计划创建与 runtime 快照查询。
- `api/tasks.py`
  任务列表查询与状态推进。
- `api/contexts.py`
  上下文包生成与提示词分段输出。
- `api/common.py`
  manifest 解析与通用 JSON 读取工具。

## `frontend/manimind-console/` 模块边界

- `src/app/`
  Next.js App Router 入口、页面壳与全局样式。
- `src/components/console/`
  控制台首页的布局壳、阶段流、任务看板和状态面板。
- `src/components/ui/`
  轻量展示组件，不承载领域状态。
- `src/data/`
  当前阶段的 mock 数据与后续 API 映射占位。
- `src/lib/`
  通用工具函数。

前端边界约束：

- 只能通过 `backend/` 提供的 HTTP API 读取项目状态。
- 不允许直接读写 `runtime/projects/` 或 `runtime/sessions/`。
- 不允许把编排规则复制进前端，编排事实仍以 `src/manimind/` 和 runtime 快照为准。

## 架构图维护粒度

- 当前 `docs/architecture.canvas` 使用模块/子系统级粒度。
- 需要入图的变更包括：
  - `frontend/manimind-console/` 的新增或其与 `backend/` 的调用边界变化
  - `src/manimind/` 新增、删除或重命名核心模块
  - 角色 owner、任务依赖或强制审核关卡变化
  - `runtime/projects/`、`runtime/sessions/`、`outputs/` 的职责变化
  - 第三方能力目录接入边界变化
- 不需要单独入图的变更包括：
  - 纯注释调整
  - 不改变模块边界的小型函数重构
  - 测试用例补充且不影响架构关系

## 推荐后续扩展

- `ingest/`
  PDF、Markdown、笔记、公式抽取。
- `memory/`
  长期上下文存储和短期协作上下文存储。
- `integrations/`
  对接 HTML skill、HyperFrames、Manim、TTS、字幕与剪辑工具。
- `review/`
  审核 Agent 规则、数学事实核查、画面一致性检查。

## 命名规则

- Python 包与模块使用 `snake_case`
- 数据模型类使用 `PascalCase`
- 配置键统一使用小写英文加下划线
- 项目 ID、镜头 ID、会话 ID 使用稳定字符串，不依赖显示标题

## 文件生命周期

- `configs/`：手工维护，长期存在
- `runtime/projects/`：项目级长期上下文，允许多次增量写入
- `runtime/sessions/`：单轮协作临时文件，可按会话清理
- `outputs/`：最终交付与可回放中间产物，默认保留
- `logs/`：运行日志，可按时间轮转
