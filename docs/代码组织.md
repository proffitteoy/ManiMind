# ManiMind 代码组织

本文是当前仓库（编排层）代码结构的简版索引，优先覆盖“可运行主链路”和“审核闭环”。

## 1. 核心目录

```text
src/manimind/             编排核心（状态机、上下文、执行器、审核闭环）
backend/                  FastAPI 接口层
frontend/manimind-console 控制台前端（/live 真实数据，/mock 骨架）
scripts/                  初始化与环境检测脚本
configs/                  manifest 示例
runtime/                  项目/会话运行时快照与事件
outputs/                  渲染产物与 approved 资产
tests/                    单元测试与链路验证样例
resources/                第三方白名单资产
```

## 2. `src/manimind` 模块

- `models.py`：阶段、任务、事件、角色等核心数据模型。
- `workflow.py`：项目计划构建、任务 DAG、角色画像、审核关卡。
- `task_board.py`：任务状态机、依赖校验、人工审核完成门禁。
- `runtime.py`：从 runtime 快照加载并派生当前阶段。
- `runtime_store.py`：`state/context/tasks/plan/events` 持久化与原子写入。
- `context_assembly.py`：按角色/阶段组装 context packet 与 prompt sections。
- `bootstrap.py`：工作区初始化、工具链探测、路径校验。
- `artifact_store.py`：按 output key 统一映射落盘路径。
- `ingest.py`：输入摄取（md/txt/pdf）。
- `worker_adapters.py`：HTML/SVG/Manim 真实执行适配器。
- `executor.py`：`run_to_review(...)`，按 `lead -> explorer -> planner -> coordinator -> workers -> reviewer(draft)` 推进到审核阶段并产出真实片段。
- `review_workflow.py`：人工 `approve` / `return` 闭环。
- `post_produce.py`：后处理与交付打包（含字幕、音频、最终视频与资产清单生成）。
- `tts.py`：TTS 适配器（`powershell_sapi` / `command` / `f5_tts` / `noop`）。
- `main.py`：CLI 命令入口（含 run-to-review/human-review/finalize）。

## 3. `backend/api` 路由

- `projects.py`：创建计划、读取项目 runtime。
- `tasks.py`：查询/更新执行任务状态。
- `contexts.py`：生成 context packet。
- `events.py`：写入结构化事件、查询项目/会话事件流。
- `execution.py`：`run-to-review` 与 `finalize` 执行入口。
- `reviews.py`：人工审核决策与 latest review-return 查询。
- `common.py`：manifest 解析、JSON/JSONL 辅助函数。

## 4. 前端页面

- `src/app/page.tsx`：默认重定向到 `/live`。
- `src/app/live/page.tsx`：读取真实 API 并展示项目概览、上下文与事件。
- `src/app/live/tasks/page.tsx`：展示任务状态、依赖与审核门禁。
- `src/app/live/review/page.tsx`：展示审核草案、审核证据与人工审核入口。
- `src/app/live/review/review-actions.tsx`：触发 `run-to-review`、`approve/return`、`finalize`。
- `src/app/live/artifacts/page.tsx`：展示产物、字幕、音频和最终视频。
- `src/app/mock/page.tsx`：保留原 mock 控制台骨架。

## 5. 运行时文件约定

- 长期上下文：`runtime/projects/<project_id>/`
- 会话上下文：`runtime/sessions/<session_id>/`
- 输出产物：`outputs/<project_id>/`
- 人工打回记录：`runtime/sessions/<session_id>/review-returns/latest.json`
- 环境检测报告：`runtime/doctor-report.json`
