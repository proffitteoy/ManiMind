# 初始化计划 A-F 执行记录（2026-05-10）

本文记录本轮针对 `docs/初始化计划.md` 的落地结果，目标是：

1. 前端可运行并可查看真实 runtime 数据。
2. 主流程可从输入推进到 `review` 阶段。
3. 审核必须人工确认，支持人工 `approve` / `return`。
4. 配音暂不接入真实服务，但预留 TTS 接口。

## 1. 已落地范围

### A. 环境基线收口
- `src/manimind/bootstrap.py`：工具探测支持显式环境变量与候选路径（含 `D:\ffmpeg\bin\ffmpeg.exe`）。
- `scripts/check-prerequisites.ps1`：同样支持 `EnvVar` 与 `CandidatePaths`，并输出探测来源。

### B. 基线固化
- 新增样例清单 `configs/max-function-review-demo.json`。
- 样例输入使用 `tests/极大函数.md`。

### C. 编排主链路收口
- 新增 `src/manimind/artifact_store.py`：统一 output key 落盘策略。
- 新增 `src/manimind/ingest.py`：支持 `md/txt/pdf` 输入摄取。
- 新增 `src/manimind/worker_adapters.py`：HTML/SVG/Manim 真实执行适配层。
- 新增 `src/manimind/executor.py`：`run_to_review(...)` 可把流程推进到 `review`，并回写真实渲染产物。
- 新增 `review.evidence` 结构化证据键，供 reviewer/human_reviewer 审核消费。

### D. 人工审核闭环
- `review.outputs` 任务 owner 改为 `human_reviewer`。
- `reviewer` 仅产出草案，不能直接放行。
- 新增 `src/manimind/review_workflow.py`：
  - `approve`：写入 `review.report` 并完成审核任务。
  - `return`：写入 `runtime/sessions/<session_id>/review-returns/latest.json`。
- `src/manimind/context_assembly.py`：`context-pack` 支持注入人工打回意见（按 `target_roles` 过滤）。

### E. 控制台接线
- 新增后端接口：
  - `POST /api/projects/run-to-review`
  - `POST /api/projects/review/decision`
  - `POST /api/projects/finalize`
  - `GET /api/projects/{project_id}/events`
  - `GET /api/projects/{project_id}/review-return`
- 前端新增 `frontend/manimind-console/src/app/live/page.tsx`：
  - 读取真实 runtime / events / review-return。
  - 支持触发 `run-to-review`、`approve`、`return`、`finalize`。
- 默认首页重定向到 `/live`，保留 `/mock` 查看原骨架页。

### F. 输入摄取与后处理
- 新增 `src/manimind/tts.py`：
  - `TTSAdapter` 协议
  - `PowerShellSapiTTSAdapter`（Windows 本地语音）
  - `CommandTTSAdapter`（外部命令注入）
  - `NoopTTSAdapter`（兜底）
- 新增 `src/manimind/post_produce.py`：
  - 审核通过后执行后处理与打包
  - 生成 `asset.manifest`
  - 推进阶段到 `done`

## 2. 新命令与示例

### CLI
```powershell
python -m manimind.main run-to-review configs/max-function-review-demo.json --session-id manual-session
python -m manimind.main human-review configs/max-function-review-demo.json approve --session-id manual-session --reason "人工审核通过"
python -m manimind.main human-review configs/max-function-review-demo.json return --session-id manual-session --reason "请修订" --must-fix "补齐术语一致性" --prompt-patch "先统一术语再生成"
python -m manimind.main finalize configs/max-function-review-demo.json --session-id manual-session --tts-provider powershell_sapi
```

### API
```http
POST /api/projects/run-to-review
POST /api/projects/review/decision
POST /api/projects/finalize
GET  /api/projects/{project_id}/runtime
GET  /api/projects/{project_id}/events?session_id=manual-session
GET  /api/projects/{project_id}/review-return?session_id=manual-session
```

## 3. 验收重点

1. `run-to-review` 后，`current_stage` 应为 `review`，并出现 `review.draft` 事件。
2. `review.outputs` 未经 `human_reviewer` 不可完成。
3. `return` 后：
   - 项目阶段应转为 `blocked`。
   - `runtime/sessions/<session_id>/review-returns/latest.json` 应存在。
   - 下一轮 `context-pack` 可读取并注入人工打回信息。
4. `approve + finalize` 后，`current_stage` 应进入 `done`，并生成 `asset.manifest`。
5. 前端 `/live` 可直接观察以上状态变化。
