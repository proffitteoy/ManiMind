# ManiMind 强前置条件约束

## 一、项目级红线

1. 不在当前仓库重复实现 `resources/skills/html-animation/`、`resources/references/hyperframes/`、`resources/skills/manim/SKILL.md` 已经提供的底层能力。
2. 不把长期上下文和短期上下文写进同一份文件或同一路径。
3. 不允许跳过审核 Agent 直接合成最终视频。
4. 不允许用隐式全局状态驱动多 Agent 协作。
5. 不允许把数学事实、论文结论、公式解释只保存在自然语言对话里而不落成结构化摘要。
6. 任何涉及模块边界、角色分布、状态存储路径、上下文装配链路或第三方能力接入方式的结构变更，必须同步更新 `docs/architecture.canvas` 与 `docs/通用项目架构模板.md`。

## 二、架构约束

1. 主 Agent 负责全局状态，不负责直接产出全部媒体细节。
2. 二级协调 Agent 负责任务切分和工具选择，不负责最终审核。
3. HTML 子 Agent、Manim 子 Agent、SVG 子 Agent 必须可以并发执行，输入输出协议要稳定。
4. 审核 Agent 必须只消费结构化产物和可追溯上下文，不能依赖“记忆里大概看过”。
5. 后处理阶段只能在审核通过后启动。

## 三、上下文约束

### 长期上下文

- 面向项目生命周期保存。
- 保存论文总结、术语表、公式清单、叙事主线、风格规范、审核结论。
- 写入路径固定在 `runtime/projects/<project_id>/`。

### 短期上下文

- 面向单次协作或单个镜头任务保存。
- 保存临时镜头草案、子 Agent 回传说明、局部失败原因、重试痕迹。
- 写入路径固定在 `runtime/sessions/<session_id>/`。

## 四、实现约束

1. 核心编排逻辑优先使用标准库和可验证的纯数据模型。
2. 在 Python 未安装前，可以先完成 PowerShell 初始化与环境检测脚本，但不能伪装成可运行的完整渲染流水线。
3. 所有环境依赖必须显式检测：`python`、`ffmpeg`、`manim`、`node`、`bun`。
4. 所有输出目录都要可预测，不能在随机临时路径下落盘。
5. 所有任务切分规则都要能从配置或代码中追溯。

## 五、外部依赖接入约束

### `resources/skills/html-animation`

- 作为 HTML 科普动画模板来源与提示词能力来源。
- 当前仓库只记录如何调用，不复制其模板内容到自有目录。

### `resources/references/hyperframes`

- 作为 HTML 预览、校验、渲染引擎来源。
- 当前仓库不在根项目内重写 HTML 渲染、帧适配、视频编码能力。

### `resources/skills/manim/SKILL.md`

- 作为 Manim 能力说明来源。
- 当前仓库负责调用契约与输入输出组织，不重写 Skill 文档中的底层流程。

## 六、交付约束

最终交付至少要包含：

- 论文总结文档
- 讲解脚本
- 分镜清单
- 任务分发表
- HTML 片段清单
- Manim 片段清单
- 审核报告
- 输出资产清单
